  Displaying Your Own Columns in the Family Browser
 

The Family Browser provides dozens of columns containing information on genes that have been computed at UCSC or have been provided by outside collaborators. In addition to these standard columns, it is also possible for users to upload their own columns for temporary display in the Family Browser. These custom columns are viewable only on the machine from which they were uploaded and are only kept for 8 hours after the last time they were accessed. Users can optionally make custom columns viewable by others as well.

Family Browser custom columns are based on files in line-oriented format. The first line of a column starts with the word "column" and describes the name, hyperlinks, and other overall characteristics of that column. This is followed by a line for each gene annotated by the column. If the file contains multiple columns, the columns are separated by blank lines. Lines starting with # are ignored.



  The Column Line
 

The first line in a column starts with the word "column", and is followed by one or more attribute=value pairs. The values must be enclosed in quotes if they contain spaces or tabs. Here is a list of the attributes:

  • name - Symbolic name (not displayed to user) for column.
  • shortLabel - Defines the column label that will be displayed at the top of the main family browser table. The default value is User Column.
  • longLabel - A short description of the column that is displayed by the name in the configuration and filter pages. The default value is Custom user column.
  • visibility - Controls if column is on by default. Put a off here to hide the column, a on here to show it. Default is on.
  • priority - Defines the order of the column relative to others. Use low numbers to make the column appear earlier in the display. Standard columns have priorities between 0 and 20. Default is 2.01.
  • itemUrl - URL to use to construct hyperlinks from column items. The %s in this string this will be substituted with the contents of the column. There is no default for this attribute.
  • labelUrl - URL of hyperlink from column label. No default.
  • org - Name of organism column describes. Default is current organism displayed by Family Browser.
  • db - Name of genome database to associate with column. This is typically something like 'hg16' or 'mm4'. Default is the current database used by Family Browser.
  • search - Set if column is searchable to one of these values:
    • exact - only match if what user puts in search box exactly matches what's in column.
    • prefix - matches if search term is same as first part of column.
    • fuzzy - matches if search term is anywhere in column.
  • idLookup - ID's are of given type. Some common types are name, swissProt, refSeq, locusLink, ensembl.
  • isNumber - If set to 'on' then filter page has numerical (min/max) controls for this column.


  Data Lines
 

The first word of a data line contains the gene ID. The rest of the line contains the value to display in the column for that gene. It is ok to have more than one line per gene. In this case the column will show a comma-separated list of values.
