Procedure for Creating a Mirror Site for Human Genome Browser


      The following procedure provides you step-by-step instructions to create a mirror browser
       incrementally. You may choose to set up either a full mirror browser or a partial one depending
      on your disk space and needs. You are also welcome to try our script which can help you to
      create a mirror site on Linux system automatically.
 

    Prerequisites:

              To set up a full mirror browser,  we recommend at a MINIMUM the following disk partitions
              and sizes:

                     MySQL Table spaces:                      ~ 50GB
                     Apache web pages area:                  ~ 40GB
                    "/projects" area for the browser:    ~ 50GB
 

    Sample Automated Script:

     We have created a script, GENOME-MIRROR.sh, which should automate the following steps
     but remains a work in progress.  It has only been tested on Redhat Linux 7.1 systems, but you may
     find it useful.
 

    Steps Full Details:

        1.  Install Apache server and MySQL server

        2.  Get all the html files and most of the text files ( 0.5 GB )

        3.  Get the data for each individual genome assembly and installing databases ( 35.1 GB )

        4.   Set up database

        5.  Get executable files
 
 

1.  Install Apache server and MySQL server

        a.    We will not provide consulting on the operation or configuration of Apache or MySQL. If you
                are not familiar with the setup of Apache or MySQL, you will have to find someone at your site
                that is. We use a reference platform of Redhat 7.1 for all the steps described here.

         b.    Find where your web pages are located.  For example on a Redhat Linux system using a
                 stock apache RPM, the web pages are stored in /var/www/html.  We will refer to this directory
                as WEBROOT in this document and you should substitute your real path for WEBROOT
                 where you see that in our writeup.

         c.    Find where your cgi-bin directory is located.  It should be under the parent directory of the
                 WEBROOT directory. For example on a Redhat Linux system using a stock apache RPM, the
                 path of the cgi-bin directory is /var/www/cgi-bin.  We will refer to this directory as CGIBIN
                 in this document and you should substitute your real path for CGIBIN where you see that.

         d.   Also find where your MySQL data is located. For example on a Redhat Linux system using stock
                RPM's, this is located in /var/lib/mysql.  We will refer to this directory as MYSQLDATA in
                this document and you should substitute your real path for MYSQLDATA where you see that.
 

2.  Get all the html files and most of the text files ( 0.5 GB )

         a.   Obtain the software package rsync, which if it isn't already installed on your system, obtain from
               http://rsync.samba.org

         b.   We are running a rsync server but you must email us first with the hostname you will be rsyncing
               from so that we can allow it access. Email to genome-mirror@cse.ucsc.edu

         c.   After we respond to your request, do a test connection:
                yourhost> rsync -navz --progress rsync://genome-rsync.cse.ucsc.edu

         d.   Determine the destination of the copy and fire off the production copy:
               The trailing slash is important!

                rsync --progress -avz --exclude='goldenPath/*/bigZips/*' \
                --exclude='goldenPath/*/chromosomes/*' --exclude='goldenPath/*/database/*' \
                --exclude='goldenPath/*/contigs/*' --exclude='goldenPath/*/pieces/*' \
                --exclude='goldenPath/archive/*' --exclude='goldenPath/bigZips' \
                rsync://genome-rsync.cse.ucsc.edu/genome/ $WEBROOT/.
 

3.   Get the data for each individual genome assembly and installing databases ( 35.1 GB )

       Use the following table to identify the freeze date ($FREEZEDATE) and the database version
              ($DBVERSION) for each genome assembly:

                        $FREEZEDATE      $DBVERSION
                          ----------         ---------
                             01apr2001                       hg7
                            12dec2000                       hg6
                             07oct2000                       hg5
                             05sept2000                 archived
                            17july2000                   archived
 

          a.   Get April 1, 2001 full data set (5.2 GB) by doing:

                i)    rsync --progress -avz   \
                       rsync://genome-rsync.cse.ucsc.edu/genome/goldenPath/01apr2001/bigZips/    \
                       $WEBROOT/goldenPath/01apr2001/bigZips/.

                ii)   rsync --progress -avz   \
                       rsync://genome-rsync.cse.ucsc.edu/genome/goldenPath/01apr2001/chromosomes/    \
                       $WEBROOT/goldenPath/01apr2001/chromosomes/.

                iii)  rsync --progress -avz   \
                       rsync://genome-rsync.cse.ucsc.edu/genome/goldenPath/01apr2001/database/    \
                       $WEBROOT/goldenPath/01apr2001/database/.

          b.   Get Dec. 12, 2000 full data set (9.4 GB),  Oct. 7, 2000 full data set (8.5 GB) and
                Sept. 5, 2000 full data set (7.2 GB) by doing:

               i)    Set the $FREEZEDATE to either 12dec2000,  07oct2000 or 05sept2000

               ii)   rsync --progress -avz   \
                       rsync://genome-rsync.cse.ucsc.edu/genome/goldenPath/$FREEZEDATE/bigZips/    \
                       $WEBROOT/goldenPath/$FREEZEDATE/bigZips/.

               iii)  rsync --progress -avz   \
                       rsync://genome-rsync.cse.ucsc.edu/genome/goldenPath/$FREEZEDATE/chromosomes/    \
                       $WEBROOT/goldenPath/$FREEZEDATE/chromosomes/.

                iv)  rsync --progress -avz   \
                       rsync://genome-rsync.cse.ucsc.edu/genome/goldenPath/$FREEZEDATE/database/    \
                       $WEBROOT/goldenPath/$FREEZEDATE/database/.

                 v)  rsync --progress -avz   \
                       rsync://genome-rsync.cse.ucsc.edu/genome/goldenPath/$FREEZEDATE/contigs/    \
                       $WEBROOT/goldenPath/$FREEZEDATE/contigs/.

          c.   Get July 17, 2000 full data set (4.8 GB) by doing:

                 i)    rsync --progress -avz   \
                       rsync://genome-rsync.cse.ucsc.edu/genome/goldenPath/17july2000/bigZips/    \
                       $WEBROOT/goldenPath/17july2000/bigZips/.

                ii)   rsync --progress -avz   \
                       rsync://genome-rsync.cse.ucsc.edu/genome/goldenPath/17july2000/pieces/    \
                       $WEBROOT/goldenPath/17july2000/pieces/.
 

 4.   Set up database

          a.  After connecting to the MySQL server, create a database called "$DBVERSION"
               corresponding to $FREEZEDATE in the table above.
                       mysql> create database $DBVERSION;

          b.  Create tables for the "$DBVERSION" database.

                i)   Move to $WEBROOT/goldenPath/$FREEZEDATE/database/ directory.
                      All the files for creating the "$DBVERSION" database are in this directory.

               ii)   Create tables:
                           % mysql -options $DBVERSION < *.sql

               iii)   Load data into the tables:
                       % mysqlimport -options $DBVERSION *.txt

           c.  Issue permission to the "$DBVERSION" database.

                You need to issue permission to allow public to access the "$DBVERSION" database.
                 mysql> grant select on $DBVERSION.* to hguser@localhost identified by "hguserstuff";
                 (In this example, public access to the "$DBVERSION" database is restricted to read-only.)

           d.  One more thing to get before your database works properly (41.7 GB).
                       rsync  --progress -avz  rsync://hgwww-dev.cse.ucsc.edu/projects/  /projects/.
 

 5.  Get executable files

            a.  Go to the CGIBIN directory, then do the following:
                        rsync  --progress -avz  rsync:'/usr/local/apache/cgi-bin/*.*'  $CGIBIN/.

            b.  Set up environment variables

                     i)   setenv  GLOBAL_CONFIG_FILE  $CGIBIN/hg.conf
                     ii)  setenv  HGCGI  $CGIBIN

             c.  Download the latest version of the files from Jim Kent's directory.
                  Follow the instructions for how to compile the files.
