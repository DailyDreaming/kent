INSTALLING
  This page is written with minor modification to Jim Kent's page.
  1. Create a directory named "jksrc" and go to it.

  2. Unzip jksrcXXX.zip, where XXX is the latest version, currently 437.

  3. Check that the environment variable MACHTYPE exists on your system.  It should on Unix. (And making this on non-Unix systems is beyond the scope of this README).  For a Linux system MACHTYPE will probably be 'i386', for Alpha it will be 'alpha', for a Sun probably 'sparc'. If necessary set up this environment variable.

  4. Make the directory ~/bin/$MACHTYPE which is where the (non-web) executables will go. Add this directory to your path.

  5. Go to the jksrc/lib directory and type make.
    On Alphas there will be some warning messages about "crudeAli.c." Otherwise it should compile cleanly under gcc.

  6. If you need to make an application in the "hg" directory do the following:

    1. If you have MySQL installed make the environment variable MYSQLINC point to where its include files are. (On my system they are at MYSQLDATA/include. Please refer to our mirror procedure page for the definition of MYSQLDATA.) While you're at it set the MYSQLLIBS variable to point to "MYSQLDATA/lib/mysql/libmysqlclient.a  /usr/lib/libz.a"

    2. If you don't have mysql installed, edit the makefile to remove 'jksql.o' and -I$(MYSQLINC).

    3. You may need to edit the makefile so that the executable files will be moved to WEBROOT/cgi-bin

    4. Type make.  It should compile with no warnings.


  7. Go to the application you want to make and type make.
    For Human Genome Browser, the most important directories that you need to compile are  hg/lib, hg/hgTracks, hg/hgc, and hg/hgBlat (in that order).
