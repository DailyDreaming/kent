INSTALLING
 
  1. Create a directory named "jksrc" and go to it.

  2. Unzip jksrc.zip

  3. Check that the environment variable MACHTYPE exists on your system.  It should on Unix. (And making this on non-Unix systems is beyond the scope of this README).  For a Linux system MACHTYPE will probably be 'i386', for Alpha it will be 'alpha', for a Sun probably 'sparc'. If necessary set up this environment variable.

  4. Go to the kent/src/lib directory and type make.
    On Alphas there will be some warning messages about "crudeAli.c." Otherwise it should compile cleanly under gcc.

  5. Go to the kent/src/jkOwnLib directory and type make.
    On Alphas there will be some warning messages about "crudeAli.c." Otherwise it should compile cleanly under gcc.

  6. For Human Genome Browser, the most important directories are in order in kent/src:
    • hg/lib
    • hg/hgTracks
    • hg/hgc
    • hg/hgBlat

  7. For each directory listed above do the following:
    1. If you have MySQL installed make the environment variable MYSQLINC point to where its include files are. For example: 
      setenv MYSQLINC /usr/local/mysql/include
--or---
setenv MYSQLINC /usr/include
      Then set the MYSQLLIBS variable to point to the location of the libmysqlclient.a library and any other libraries needed on your platform to link network applications. 
      setenv MYSQLLIBS '/usr/local/mysql/lib/mysql/libmysqlclient.a'

---Solaris for example often needs----

setenv MYSQLLIBS '/usr/local/mysql/lib/mysql/libmysqlclient.a -lnsl -lsocket -lresolv'
    2. You may need to edit the makefile so that the executable files will be moved to WEBROOT/cgi-bin and not our internal paths. This will be fixed in a future release.
    3. cd into the directory listed.
    4. Type make.  It should compile with no warnings.
