Description

The UCSC Genes track shows gene predictions that are based on data from RefSeq, Genbank, and UniProt. The gene predictions are a moderately conservative set, requiring the support of one GenBank RNA sequence plus at least one additional line of evidence. RefSeq RNAs are an exception to this, and require no additional evidence. The track includes both protein coding and uncoding transcripts. Some of the uncoding transcripts may actually code for protein, but the evidence for the associated protein is at best weak. In general this gene set has about 10% more protein coding genes, about 5x as many uncoding genes, and about 2x as many splice varients as RefSeq.

Display Conventions and Configuration

This track in general follows the display conventions for gene prediction tracks. For uncoding genes all of the exons are relatively thin, while for coding genes the coding exons are relatively thick. Black coloring indicates features that have corresponding entries in the Protein Databank (PDB). Dark blue indicates that the transcript has been reviewed or validated, either by the RefSeq or the SwissProt staff. Medium blue is used for other RefSeq transcripts. Non-RefSeq transcripts are colored light blue.

This track contains an optional codon coloring feature that allows users to quickly validate and compare gene predictions. To display codon colors, select the genomic codons option from the Color track by codons pull-down menu. Click here for more information about this feature.

Methods

The UCSC Genes are built in a multistep pipeline. The major steps are:

  • Align RefSeq and GenBank RNA to genome with BLAT, keeping only the best alignments for each RNA, and discarding alignments of less than 98% identity.
  • Break up alignments at non-intronic gaps, and throw out small isolated fragments.
  • Create a splicing graph for each set of overlapping alignments. This graph has an edge for each exon or intron, and a vertex for each splice site, start, and end. Each RNA that contributes to an edge is kept as evidence for that edge.
  • Create a similar splicing graph in the mouse based on mouse RNA and ESTs. If the mouse graph has an edge that is orthologous an edge in the human graph, add that to the evidence for the human edge.
  • If an edge in the splicing graph is supported by two or more human ESTs, add that as evidence for the edge.
  • Traverse the graph to generate all unique transcripts. The traversal is guided by the initial RNAs to avoid a combinatorical explosion in alternative splicing. Only transcripts that include an edge supported by one additional line of evidence past the RNA are output. For single exon genes either two RNAs or two additonal lines of evidence are required. A RefSeq RNA however is by itself enough.
  • Generate protein predictions. For non-RefSeq transcripts we use the txCdsPredict program to determine if the transcript is protein coding and if so where the start and stop codons are. The txCdsPredict weighs the length of the protein, the presence of a Kozak consensus sequence at the start codon, and the length of the orthologous predicted protein in other species as positive evidence. As negative evidence it considers nonsense-mediated-decay, and start codons in any frame upstream of the predicted start codon. For RefSeq transcripts we use the RefSeq protein prediction.
  • Find corresponding UniProt protein if any.

Credits

The Known Genes track was produced at UCSC using a computational pipeline developed largely by Jim Kent. It is based on data in NCBI RefSeq, UniProt (including TrEMBL and TrEMBL-NEW) and GenBank. Our thanks to the people running these databases and to the scientists worldwide who have made contributions to them.
