Using the ENCODE DCC Submission Site

Data is submitted to the ENCODE DCC as a compressed archive of files. This archive consists of data files, together with metadata files describing the data files. Each ENCODE project will work with the ENCODE DCC to create a data agreement that specifies the data file and metadata file formats for their project.

Once the data agreement is finalized, data is submitted as follows:

  • Create a project
  • Submit the submission archive to the project
  • Verify that the submission was accepted
  • When notified by the DCC, approve the track(s) in the browser
If the project submission fails, create an archive containing corrected versions of any problem files, and submit this to the project. If any failure messages are unclear, contact the DCC for assistance.
